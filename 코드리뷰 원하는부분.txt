3. naverController에 네이버 소셜 로그인 기능을 구현했었습니다. 근데 저는 통신 모듈을 이용해 네이버에서 제공하는 restful api를 이용했는데 passport로도 많이 구현을 하던데 어떤 상황에서 어떤 방법이 더 좋은지 알고 싶습니다.


4. chatController에 있는 채팅 관련 api 질문입니다. 보통 채팅을 구현할 때 firebase에서 지원하는 기능같이 다른 곳에서 구현하기 쉽게 도와주는 방식을 많이 이용했던데, 저는 api를 쌩으로 구현했었습니다. 아마 제대로 된 채팅을 받으려면 클라이언트 측에서 일정 시간(눈치채기 힘들 만큼) 마다 계속 서버와 통신을 해야 제대로 된 채팅 기능이 구현이 될 것 같은데, 이런 방식이 맞는지도 궁금합니다.


5. searchController 검색 기능을 구현할 때 쿼리를 %~~% 이런 식으로 작성했는데, 보통 앞뒤로 %를 붙이는 방식은 성능에 크게 문제가 생겨 별로 선호하지 않는다는 말을 들었습니다. 근데 MySQL에서 % % 구문을 사용하지 않고 검색을 할 수 있는 방법이 없는 거 같아서 그냥 다들 % % 방식으로 쓰는지 아니면 검색 서버를 따로 만들어서 엘라스틱 서치 같은 것을 이용하는지 궁금합니다.
